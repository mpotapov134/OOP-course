Необходимы классы:
1. Universe - хранит состояние вселенной (имя, правила, координаты клеток) и
   рассчитывает состояние через n итераций. Должен быть метод, печатающий
   вселенную в консоль, и метод, записывающий вселенную в файл.
2. Reader - читает состояние вселенной из файла.
3. Writer - записывает состояние вселенной в файл.
4. ArgumentParser - обрабатывает параметры командной строки. Хранит
   map<string, string> (параметр, значение). Должен быть метод, читающий все
   параметры и сохраняющий их значения. Если значения некорректны, бросает
   исключение. Если переданы недопустимые параметры, бросает исключение. Должен
   быть метод, проверяющий корректность набора параметров. Должен быть метод,
   возвращающий мапу параметров.
5. ComandHandler - обрабатывает команды во время игры. Должен быть метод,
   парсящий и исполняющий команду. Если команда некорректна, сообщить об этом
   пользователю и вывести справку в консоль. Исполнение должно быть разбито на
   методы для каждой команды.

Чтение входного файла.

Возможность открывать файл в конструкторе и отдельным методом. Конструктор по
умолчанию ничего не делает.
Метод для проверки, открыт ли файл.
Методы, читающие и возвращающие имя вселенной, правило перехода и список
координат клеток.
Возможность закрыть файл отдельным методом. Файл должен быть закрыт в
деструкторе.

1. Открытие файла: если уже есть открытый файл, закрыть его и открыть новый.
   Если не удалось открыть, бросить исключение.
   Проверить формат файла, если не Life 1.06, бросить исключение.

2. Чтение: перейти на соответствующую позицию в файле, прочитать данные,
   вернуть в удобном формате (строка для имени, set<int> для правил,
   set<pair<int, int>> для координат). Если отсутствует имя и/или одно/оба
   правила, вывести сообщение об ошибке и вернуть значения по умолчанию.

Запись в выходной файл.

Возможность открывать файл в конструкторе и отдельным методом. Конструктор по
умолчанию ничего не делает.
Метод для проверки, открыт ли файл.
Метод, принимающий объект Universe и записывающий вселенную в файл в формате
Life 1.06.
Возможность закрыть файл отдельным методом. Файл должен быть закрыт в
деструкторе.

Обработка параметров командной строки.

1. Чтение параметров: проходим по массиву параметров, переданных программе при
   запуске. Определяем имя и значение очередного параметра. Если имя
   некорректно, бросаем исключение с сообщением об ошибке. Если значение не
   было передано или некорректно (для -i, --iterations), бросаем исключение с
   сообщением об ошибке. Если параметр уже встречался, бросаем исключение с
   сообщением об ошибке. Записываем значение параметра в мапу по краткому имени.
2. Проверка корректности набора параметров: допустимые наборы: {без параметров;
   имя входного файла; имя входного файла, число итераций, имя выходного
   файла}. Составляем set<string> имен параметров и сравниваем с допустимыми.
   Если не совпадает ни с одним допустимым набором, бросаем исключение с
   сообщением об ошибке.

Обработка команд во время игры.

Парсим команду. Если такой команды не существует или аргумент команды
некорректен, выводим сообщение об ошибке. Иначе вызываем соответствующий метод
исполнения для данной команды.
